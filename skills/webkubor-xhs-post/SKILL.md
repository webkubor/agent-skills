# webkubor 小红书技术文案

> 基于小红书爆款笔记 5 要素（选题/文案/标题/封面/CTA），干货+情绪混合。
> 封面走 `enterprise-tech-cover` skill。本 skill 负责正文。

## 账号定位

| 维度 | 内容 |
|------|------|
| 账号类型 | 个人 IP 类 |
| 目标用户 | AI 时代的开发者、技术人、独立开发者 |
| 核心痛点 | AI 工具眼花缭乱，不知道什么真正有用 |
| 价值主张 | 不教工具，分享真实思考——Agent 会过时，判断力不会 |
| 一句话 | AI 时代下，我们真正该沉淀什么？ |

## 笔记类型：干货 + 情绪混合

webkubor 不走纯教程路线。每篇都同时做两件事：
- **干货**：真实踩坑经验、可操作的方法
- **情绪**：个人观点、共鸣场景、反常识判断

> 公式：情绪负责拉住人，干货负责留下人。

## 标题公式

> `目标用户 + 具体场景/问题 + 承诺结果 + 角度`

| ❌ 弱标题 | ✅ 强标题 |
|-----------|-----------|
| Agent 协作经验分享 | Agent 记忆是靠不住的 |
| AI 工具使用心得 | 文档让 AI 自觉。硬门槛让你放心 |
| 前端开发自动化实践 | 我已经很久没写代码了 |

**原则**：
- 一眼看懂和自己有什么关系
- 传达利益、冲突、反常识或结果
- 不制造过高预期，和内容匹配

## 文案结构：危机化解型

```
开头钩子（1-2句，直接给冲突/场景）
    ↓
问题展开（2-3句，发生了什么）
    ↓
根因分析（逐条，每条一个具体判断）
    ↓
解决方案（不泛泛而谈，落到具体动作）
    ↓
金句收尾（1句，能独立传播）
    ↓
评论区引导 + 标签
```

**原则**：
- 开头尽快进入主题，不铺垫
- 主体持续给信息或观点
- 短段落，一句一段
- 干货里加入个人故事

## 文案语气

- 第一人称「我」
- 口语化，像跟同行聊天
- 不居高临下，不说教
- 不编场景，不用假例子
- 禁止 AI 味词汇（「在…背景下」「随着…的发展」）

## CTA（行动号召）

- 评论区引导：问一个问题，让读者想说点什么
- 收藏理由：这条内容以后能用到

## 标签策略

技术标签 + 场景标签 + 人群标签，3-5 个。

图文发帖，封面第一张，后面 4-5 张支撑正文逻辑。
