# webkubor 小红书文案 QA

> 不是写作模板。是发布前的质量检查关。不通过 = 不发。

## 七道 QA Gate

每篇文案推给栖洲前，逐项过。任一不通过 → 打回改。

### Gate 1：选题立意

- 这篇到底要说什么？一句话能说清楚吗？
- 是基于真实经历，还是编的场景？
- 这个观点只有栖洲能说吗？（不是任何人用 AI 都能写的）
- 有没有具体的冲突/问题场景？还是空谈方法论？

### Gate 2：逻辑严密

- 问题 → 根因 → 解决，这条链完整吗？
- 有没有跳过中间步骤？（比如「文档不可靠」直接跳到「加硬门槛」——中间缺了「为什么文档不可靠」的分析）
- 每个「因为」后面都有「所以」吗？

### Gate 3：词语精准

- 有没有引入原文没出现的概念？
- 「自觉」「可靠」「硬门槛」——每个词的出现，栖洲原文里有没有？
- 有没有为了节奏好看而塞进去的词，实际意思不对？

### Gate 4：歧义检测

- 每句话只有一种读法吗？
- 有没有模棱两可的表达？
- 「文档让 AI 自觉」——什么意思？是「AI 会主动遵守文档」还是「AI 在假装遵守」？原文说清楚了吗？

### Gate 5：逻辑冲突

- 前面说的和后面说的矛盾吗？
- 同一个概念前后用词一致吗？（前面叫「硬门槛」，后面不能变成「硬检查」「硬约束」）
- 开头的问题场景，结尾回应了吗？

### Gate 6：立意收束

- 最后一句话落地了吗？还是飘了？
- 收尾金句是全文的结论，还是另外编了一句漂亮话？
- 读者读完最后一句，能一句话转述给同事吗？

### Gate 7：人设一致

- 第一人称「我」？
- 是「分享经验」还是「教别人」？
- 有没有居高临下的语气？
- 有没有 AI 味？（「在…背景下」「随着…的发展」「不仅…而且…」）

## 复盘维度（发布后）

数据回来之后，按这四层复盘：

| 指标 | 看什么 |
|------|--------|
| 点击率低 | 标题 + 封面 |
| 读完率低 | 开头 + 结构 + 节奏 |
| 互动弱 | 内容有没有观点？有没有引发共鸣？ |
| 转粉弱 | 定位清晰吗？内容持续指向同类人群？ |
