# 质量审查 Skill

> **管线位置**: 最后一站 — 生成前的最终审查 + 生成后的问题诊断
> **触发**: prompt 编译完成 / 生成结果有问题
> **关联**: 生成问题归入 `references/failure-atlas.md`

---

## 一、生成前审查（Prompt Review）

### 8 维度审查清单

#### 1. 主体检查
- [ ] 是否定义清楚（有特征描述或参考图绑定）
- [ ] 标签是否持续一致使用
- [ ] 多主体场景是否有独立的标签区分

#### 2. 分镜结构
- [ ] 是否按「镜头1/2/3」拆分
- [ ] 每个镜头是否有明确的叙事功能
- [ ] 分镜间是否有因果推进

#### 3. 动作描述
- [ ] 是否包含肢体部位
- [ ] 是否量化了速度/力度/幅度
- [ ] 动作链是否连贯

#### 4. 运镜指令
- [ ] 每个镜头是否不超过一个主运镜
- [ ] 运镜是否与场景情绪匹配

#### 5. 光影氛围
- [ ] 是否指定了光源类型和方向
- [ ] 相邻镜头光影是否一致
- [ ] 色温描述是否明确

#### 6. 声音设计
- [ ] 台词是否标注了角色标签和情绪
- [ ] 环境音和动作音效是否覆盖
- [ ] 音乐情绪是否匹配

#### 7. 风格锚定
- [ ] 是否包含风格关键词（参见 style-dictionary）
- [ ] 是否与项目整体基调一致

#### 8. 约束条件
- [ ] 是否包含对应的 negative prompt
- [ ] 平台硬限制是否合规（时长/文件数/格式）

## 二、生成后诊断（Result Review）

### 诊断流程

```
生成结果 → 发现具体问题 → 对照 failure-atlas.md 找匹配项
    ↓ 无匹配
记录新问题 → 分析根因 → 修复 → 重试
```

### 快速分诊表

| 症状 | 最可能原因 | 优先级 |
|------|-----------|--------|
| 画面抖动 | 运镜冲突 / prompt 太复杂 | P0 |
| 人脸变化 | 未用 image-to-video / 参考图差 | P0 |
| 动作跳帧 | 分镜过渡不充分 | P0 |
| 文字自动出现 | 未加负面约束 | P1 |
| 色调不统一 | 未锚定色彩关键词 | P1 |
| 画面劣化 | 续写次数过多 | P1 |
| 动作僵硬 | 缺少肢体细化 | P2 |
| 节奏不佳 | 分镜时长分配不合理 | P2 |

## 四、迭代策略

### 一次只改一个大变量
如果生成失败，从以下只选一个维度修改：
1. 提示词文案
2. 参考素材选择
3. 运镜指令
4. 时长
5. 画幅比例

### 缩短原则
如果画面不稳定，优先缩短范围：
1. 减少动作数量
2. 减少道具/元素
3. 简化运镜
4. 收紧画框（中景→近景）
