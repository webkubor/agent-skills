# Seedance 2.0 — 视频引擎使用规范

> **模型**: Seedance 2.0（字节跳动即梦平台）
> **定位**: 本体系中所有动态视频生成的唯一引擎
> **平台**: jimeng.jianying.com / Volcengine Ark API
> **参考**: 官方文档 https://www.volcengine.com/docs/82379/2222480

---

## 一、三种生成模式

| 模式 | 输入 | 适用场景 | 关键约束 |
|------|------|----------|----------|
| **Text-to-Video** | 纯文字 | 探索创意、无参考素材 | 先写主体，动作用物理词，运镜简单 |
| **Image-to-Video** | 图片 + 文字 | 首帧/产品/构图已存在 | 尽早引用图片，只描述一层运动，不变更主体身份 |
| **Reference-to-Video** | 视频参考 | 一致性优先于创意自由 | 先写什么必须固定，再写轻量运动指令 |

---

## 二、提示词进阶公式

```
精准主体 + 动作细节 + 场景环境 + 光影色调 + 镜头运镜 + 视觉风格 + 画质 + 约束条件
```

### 1. 定义主体
```text
将 <图片1> 中的 [2-3个静态特征] 定义为 <主体1>
```
- 特征要求：服饰、发型、外观、类别等清晰稳定的特征
- 多主体需分别定义，使用稳定标签
- 每次提及主体时用同一标签，避免指代混乱

### 2. 分镜时序
按事件发生顺序，用「镜头1 / 镜头2 / 镜头3」拆分：
```text
镜头1：[运镜] + [主体动作] + [空间位置] + [音频]
镜头2：[运镜] + [主体动作] + [空间位置] + [音频]
```
- 不强制限制每段时长，让模型自然生成节奏
- 每个镜头按：运镜 → 动作 → 位置 → 音频 组织

### 3. 动作描述
- 肢体细化：具体到手、腿、头部、肩背
- 程度量化：幅度、速度、力度（缓慢抬手、快速转头、用力蹬地）

---

## 三、分镜时序模板

### ≤15 秒单段视频
```text
[时长]秒 [主题]，[0-N秒]：[运镜] + [主体] + [动作] + [场景]；
[N-M秒]：[运镜切换] + [主体] + [动作]；[M-结尾秒]：[收束镜头]
```

### >15 秒多段拼接
```text
## 超长视频提示词（总时长约XX秒）
### 第1段（0-N秒）
**生成时长**：N秒
**提示词**：...
**衔接点**：本段结尾画面：[描述]

### 第2段（N-M秒）
**操作**：将第1段生成的视频上传为 @视频1
**提示词**：将@视频1延长M-N秒。...
```

## 四、多模态参考系统

| 素材类型 | 语法 | 数量上限 | 用途 |
|----------|------|----------|------|
| 图片 | `@图片1` ~ `@图片9` | 9张 | 人物参考、场景背景、首尾帧 |
| 视频 | `@视频1` ~ `@视频3` | 3个 | 运镜、动作、特效、节奏 |
| 音频 | `@音频1` ~ `@音频3` | 3个 | 背景音乐、语音音色、音效 |

- 总文件上限 12 个
- 图片: jpeg/png/webp/bmp/tiff/gif, 单张 <30MB
- 视频: mp4/mov, 2-15秒, 单段 <50MB, 480p-720p
- 音频: mp3/wav, 总时长 ≤15秒, 单段 <15MB

## 五、运镜语言速查

| 运镜类型 | 英文 | 中文示例 |
|----------|------|----------|
| 推拉变焦 | dolly zoom / push in / pull out | 镜头缓慢推向主体面部 |
| 跟踪镜头 | tracking shot / follow | 侧面跟踪镜头跟拍人物行走 |
| 航拍平移 | aerial pan / drone shot | 无人机从城市上空缓慢平移 |
| 环绕旋转 | orbit / 360 rotation | 围绕主体缓慢旋转180度 |
| 手持晃动 | handheld / shaky cam | 手持镜头向前跟拍，轻微晃动 |
| 固定机位 | static / locked-off | 固定机位，人物从左入画 |
| 仰拍 | low angle | 仰拍特写，角色居高临下 |
| 俯拍 | overhead / bird's eye | 俯拍城市街景，逐渐拉远 |
| 快速摇镜 | whip pan / fast pan | 镜头快速摇动切到特写 |
| 慢镜头 | slow motion | 慢动作捕捉，雨水滴落的瞬间 |

## 六、Negative Prompt（负面约束）

**每个 prompt 必须包含 negative prompt**，按场景选择：

### 通用
```text
no shaky camera, no object melting, no random text, no muddy lighting,
no watermark, no subtitles, no logo
```

### 电商/产品
```text
no logo distortion, no text artifacts, no packaging collapse,
no duplicate product, no label blur
```

### 人物/UGC
```text
no extra fingers, no face drift, no lip mismatch, no background warping,
no plastic skin, no anime face
```

### 电影感
```text
no shaky camera, no object melting, no random text, no muddy lighting,
no extra limbs, no deformation
```

## 七、迭代规则

1. **一次只改一个大变量**：prompt文案 / reference选择 / 运镜指令 / 时长 / 画幅 — 只选一个改
2. **不稳定时缩短范围**：减少动作数量、减少道具、简化运镜、收紧画框
3. **根据 prompt 判断模式**：如果反复提到"同一张脸""同一瓶产品""同一形状"，切换为 image-to-video

---

## 八、平台限制

- 单次生成 4-15 秒
- 最高 2K 分辨率
- 不支持包含真实人脸的照片/视频上传（会被自动拦截）
- 多次续写会叠加劣化（人脸区域出现斑驳色块）
- 视频延长操作格式：`将@视频1延长N秒`（不要写"参考@"）

---

## 九、质量检查清单

- [ ] 主体定义清晰（使用标签持续指代）
- [ ] 分镜逻辑合理（镜头1→2→3 有因果推进）
- [ ] 动作描述有肢体细化+程度量化
- [ ] 包含运镜指令
- [ ] 包含负面约束
- [ ] 超过15秒的视频做了多段拼接计划
- [ ] 参考素材格式合规
- [ ] 风格锚定与项目基调一致（参见 wuxia-cinematic）
