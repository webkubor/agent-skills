# 故障诊断库

> 记录本体系所有已知的生成失败模式、根因与修复方案。
> 每次遇到新的失败模式，追加一条记录。

---

## 图片故障（gpt-image-2）

### P-F-001 文字渲染错误
**症状**: 图片中文字缺失、多字、错字或变形
**根因**: 未使用 High 模式；中文文字指令不够精确
**修复**:
1. 切换到 High 模式
2. 用引号包裹所有文字
3. 指定文字位置、颜色、字号
4. 减少文字数量（核心关键词优先）
5. 考虑后期用设计工具添加文字

### P-F-002 风格漂移
**症状**: 同一系列不同图片风格差异大
**根因**: 每次 prompt 的风格描述不同
**修复**:
1. 使用固定的风格锚定模板（参见 style-dictionary）
2. 将风格段落固定为 prompt 尾部不变部分
3. 只修改 Subject 和 Composition 部分

### P-F-003 肢体/手指变形
**症状**: 人物手指数量不对、肢体扭曲
**根因**: 默认模式下的常见问题
**修复**:
1. 切换到 High 模式
2. 增加 negative prompt: `no extra limbs, no deformed hands, no missing fingers`
3. 减少画面中人物数量
4. 选择手部不显眼的构图方式

### P-F-004 构图偏移
**症状**: 主体位置偏离预期
**根因**: 构图指令模糊
**修复**:
1. 明确指定主体占画面百分比
2. 使用精确构图词：「居中俯拍」「左侧三分之一处」「右下角」
3. 给出画幅比例

### P-F-005 多元素混乱
**症状**: 画面元素过多，主次不分
**根因**: prompt 中描述了太多元素
**修复**:
1. 减少元素数量：一次只描述 1-3 个核心元素
2. 明确主体优先级
3. 使用「简洁背景」「大量留白」指令

## 视频故障（seedance 2.0）

### V-F-001 画面抖动/不稳定
**症状**: 视频画面抖动、物体跳变
**根因**: 运镜指令冲突；单 prompt 包含太多动作
**修复**:
1. 简化运镜：每次只用一个运镜方式
2. 添加 negative: `no shaky camera, no jitter`
3. 缩短时长（先试 5-8 秒）
4. 减少画面中的主体/道具数量

### V-F-002 人脸漂移/身份丢失
**症状**: 人物面部特征在视频中变化
**根因**: 未使用 image-to-video 模式；参考图不清晰
**修复**:
1. 切换为 image-to-video 模式
2. 使用高清正脸参考图
3. 添加约束: `face identity consistent, no face drift`
4. 对同角色使用固定标签持续指代

### V-F-003 画面劣化（多次延长后）
**症状**: 多次延长后画面出现斑驳色块，人物面部模糊
**根因**: 模型续写会叠加劣化
**修复**:
1. 控制续写次数（建议 ≤2 次）
2. 将原始视频转为白模视频后再续写
3. 白模 prompt: `将视频转为白色3D模型，纯白背景，无色彩无纹理，运动流畅`
4. 优先用高清图片做参考素材

### V-F-004 审核拦截
**症状**: 生成被平台拦截（非内容违规类）
**根因**: 提示词触发了过度过滤
**修复**:
1. 检查是否包含受保护 IP 名称/公众人物姓名
2. 使用 seedance-copyright 技能做安全改写
3. 检查参考素材是否含真实人脸
4. 用更中性的词汇替换可能敏感的表述

### V-F-005 动作不连贯
**症状**: 动作之间有明显跳帧
**根因**: 分镜过渡描述不充分；动作变化过大
**修复**:
1. 增加分镜间的过渡描述
2. 控制相邻分镜的动作幅度
3. 用「流畅过渡」「无缝衔接」指令
4. 减少总动作数量

### V-F-006 字幕/水印自动出现
**症状**: 不需要字幕时画面出现文字
**根因**: 模型默认倾向
**修复**:
1. 添加明确约束: `保持无字幕，避免生成任何文字或字幕`
2. 使用横屏模式（横屏出现字幕概率更低）
3. 去除参考素材中的文字

### V-F-007 风格不一致
**症状**: 同系列视频风格差异大
**根因**: 未使用统一的风格锚定
**修复**:
1. 固定风格关键词（参见 style-dictionary）
2. 使用参考视频做风格锁定
3. 每段视频复用相同的风格描述段落

## 通用故障

### G-F-001 Prompt 过长导致无效
**症状**: 长 prompt 反而生成质量差
**根因**: 信息过载，模型无法聚焦
**修复**:
1. 缩短至 100-200 字内
2. 删除冗余形容词
3. 分多轮生成（先生成基础版本→编辑细化）

### G-F-002 多次生成结果不稳定
**症状**: 同样 prompt 两次生成差异大
**根因**: 模型随机性
**修复**:
1. 生成 2-3 个版本选最优
2. 使用更确定的描述（减少「可能」「大概」等模糊词）
3. 用参考图增强确定性
