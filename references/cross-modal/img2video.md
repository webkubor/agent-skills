# 图→视频串联工作流

> **定位**: 将 gpt-image-2 生成的图片无缝交给 Seedance 2.0 做视频生成的流程指南
> **适用**: 产品展示、MV、社交媒体短片等需要先有图再有视频的场景

---

## 工作流概览

```
gpt-image-2 生成首帧图 → 导出为 Seedance 兼容格式 →
加载为 @图片1 → Image-to-Video 模式 → 生成视频
```

---

## 工作流详情

### 步骤 1：确定首帧

在 gpt-image-2 中生成符合 Seedance 输入要求的图片：

```yaml
格式: jpeg / png / webp
尺寸: ≤ 30MB
内容: 不含真实人脸（Seedance 平台限制）
构图: 预留运动空间（主体不要太靠边）
画质: 高清、清晰、不模糊
```

### 步骤 2：首帧构图预留

为 Image-to-Video 模式留出运动空间：

| 预期运动方向 | 首帧构图策略 |
|-------------|-------------|
| 主体向左移动 | 主体偏右 |
| 主体向右移动 | 主体偏左 |
| 360度旋转 | 主体居中 |
| 推近特写 | 主体偏小（推近后放大） |
| 拉远全景 | 主体偏大（拉远后缩小） |
| 自上而下 | 主体偏上 |
| 自下而上 | 主体偏下 |

### 步骤 3：编译 Image-to-Video Prompt

Image-to-Video 的完整 prompt 结构：

```text
@图片1 [主体引用]，[一层运动描述]，[运镜指令]，
[光影风格]，[画质]，[约束条件]，[负面约束]
```

### 步骤 4：多段串联

如果生成多个视频段，保持首帧的视觉连续性：

```
段1：Image-to-Video 生成（首帧 = 段1首帧）
段2：延长段1，保持主体一致
段3：延长段2，或新的 Image-to-Video（新首帧 = 段2结尾帧）
```

## 典型场景串联

### 场景 1：产品展示视频

```
1. gpt-image-2 生成产品正面标准图（白底/影棚光）
2. Image-to-Video: 360度产品旋转展示
3. 延长: 细节特写（镜头推近到 logo/质感）
4. 延长: 使用场景（产品置于生活场景中）
```

### 场景 2：武侠角色登场

```
1. gpt-image-2 生成角色站立全景图（wuxia-cinematic 风格）
2. Image-to-Video: 角色从静止到缓慢转身面对镜头
3. 延长: 镜头推近面部特写
4. 延长: 角色拔剑出鞘动作
```

### 场景 3：社交媒体短片

```
1. gpt-image-2 生成 lifestyle 场景图（social-cover 风格）
2. Image-to-Video: 微小视差运动 + 手持呼吸感
3. 延长: 场景切换（新图 → 新 Image-to-Video）
```

## 跨模态一致性备忘

- gpt-image-2 和 Seedance prompt 中使用 **相同的风格锚定词**
- 色彩关键词从 `references/style-dictionary.md` 中选用
- 光影描述保持一致（光源方向、色温、对比度）
- 人物面部描述保持一致（参见 `cross-modal/brand-consistency.md`）
